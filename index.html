Recommandations · Board 2026

Priorités pour la mise en place.

Six actions prioritaires pour que les 3 Centres de Compétences IA soient opérationnels et générateurs de revenus d'ici fin 2026.

01 · Priorité immédiate
Nommer les 6 leaders CDI en Q1 2026
Sans les chefs de Centre et leads CDI, les Centres restent des groupements informels sans P&L, sans doctrine et sans capacité à recruter. C'est la condition structurelle de tout le reste.
Action : Recrutement immédiat
02 · Différenciation marché
Concentrer le Centre 02 sur l'IA Agentique
L'IA Générative se commoditise. L'avantage concurrentiel d'AVA2i sur 18-24 mois est l'IA Agentique — LangGraph, MCP, multi-agents. Le Centre 02 doit être la référence française.
Focus : Agentique 2026
03 · Revenus récurrents
Faire du Centre 03 le pivot de l'ARR
AVAshield et Diplomat sont les seules sources d'ARR prévisible. Chaque mission de conseil doit générer une opportunité produit. Objectif : 200k€ ARR 2026, 800k€ en 2027.
Objectif : 200k€ ARR 2026
04 · Synergie partenaires
Aligner Centre 01 avec Snowflake & Databricks
Le Centre 01 est le point d'entrée du co-selling. Chaque deal partenaire passe par lui. Objectif : Snowflake Premier Partner Q3 2026 grâce aux certifications et projets référencés.
Cible : Snowflake Premier Q3
05 · Acquisition par la preuve
2 audits de maturité IA par semaine dès Q2
Le Centre 03 est responsable des audits IA — outil d'entrée commercial d'AVA2i. Chaque audit génère un pipeline pour les Centres 01 et 02. Le Centre 03 est le moteur commercial des deux autres.
KPI : 2 audits / semaine
06 · Gouvernance
Comité des Centres mensuel
Les 3 Chefs de Centre se réunissent mensuellement pour synchroniser les pipelines, partager les méthodologies et allouer les ressources entre Centres. Mécanisme anti-silos critique.
Gouvernance : Mensuel

⚙ Configurer Firebase — Collab temps réel

Pour activer la synchronisation en temps réel avec ton binôme, crée un projet Firebase gratuit et remplace les valeurs dans le code.
Étape 1 — Créer le projet
  1. Va sur console.firebase.google.com
  2. Clique "Ajouter un projet" → nomme-le ava2i-roadmap
  3. Dans Realtime Database → Créer une base → Mode test (30 jours)
  4. Dans Paramètres du projet → "Ajouter une application Web" → copie le firebaseConfig
Étape 2 — Coller la config dans le fichier HTML
Ouvre AVA2i_Centres_IA_V2.html dans un éditeur de texte.
Cherche const FIREBASE_CONFIG et remplace les valeurs
"VOTRE_API_KEY", "VOTRE_PROJECT", etc. par les vraies valeurs copiées.
Étape 3 — Publier sur GitHub Pages
Upload le fichier modifié dans ton repo GitHub → Settings → Pages → branche main.
Partage l'URL GitHub Pages avec ton binôme. Chaque modification est visible en temps réel pour tous les collaborateurs connectés.
💡 Gratuit — Firebase Realtime Database offre 1 Go de stockage et 10 Go/mois de transfert. Largement suffisant pour cette roadmap.
